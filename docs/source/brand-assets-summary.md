## Brand Positioning

Knowledge OS transforms organizational knowledge into actionable context. It should present as an enterprise intelligence system, not a graph database, documentation browser, search engine, data catalog, or visualization tool.

Primary attributes:

- Intelligent.
- Structured.
- Confident.
- Calm.
- Technical.

Secondary attributes:

- Insightful.
- Modern.
- Operational.
- Precise.
- Trustworthy.

Avoid:

- Playful.
- Consumer-focused.
- Startup gimmicky.
- Crypto-inspired.
- Gaming-oriented.
- Cyberpunk.
- Experimental.

## Logo Notes

The source logo is a horizontal Knowledge OS mark on a cool gray-blue gradient background. It includes a networked cube/graph icon and a white-to-blue wordmark. The accent blue in the guide is derived from the logo.

Do not recreate the logo in code. Use the approved bitmap asset unless a future vector asset is supplied.

## UX Commitments

- The graph is invisible.
- Understanding beats visualization.
- Project Spaces are the primary destination artifact.
- ChatGPT Projects, GPTs, SharePoint, Copilot, and similar tools can be primary execution runtimes.
- Every major page should support Explain, Summarize, Explore, and Generate.
- Preferred visualizations: architecture layers, dependency trees, coverage heatmaps, and relationship tables.
- Avoid force-directed graphs, hairball diagrams, 3D graphs, animated network clouds, and complex node-link views.
