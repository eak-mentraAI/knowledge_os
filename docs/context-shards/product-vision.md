# Product Vision Shard

Knowledge OS is an enterprise reasoning platform.

Users should feel like they are teaching the platform a domain and creating useful project workspaces. They should not feel like they are operating a graph database.

Primary outputs:

- Knowledge graph.
- Embedding layer.
- Context packs.
- Project spaces.
- Assistant configurations.
- Context delivery packages for ChatGPT Projects, GPTs, SharePoint, Copilot, and markdown repositories.

Primary navigation:

- Dashboard.
- Sources.
- Knowledge Model.
- Domains.
- Project Spaces.
- Assistants.

Primary UX goal:

- Help users understand a domain and create a workspace for solving a problem.

Positioning:

- Knowledge OS is the compiler and intelligence layer.
- GPT Projects, GPTs, SharePoint, Copilot, and similar environments are execution runtimes.
- The project space is the artifact; the runtime is where collaboration and delivery happen.

Non-goals for MVP:

- Automatic diagram generation.
- Source-system mutation.
- Synthetic documentation creation.
- Enterprise search replacement.
- Automatic ontology generation.
