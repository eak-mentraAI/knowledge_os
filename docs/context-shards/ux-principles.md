# UX Principles Shard

Users should never feel like they are building a graph.

Product principles:

- Hide graph complexity.
- Prioritize understanding over visualization.
- Make Project Spaces the primary destination.
- Treat external runtimes as the primary execution environments when that best fits the user's workflow.
- Make AI actions contextual and useful.

Every major screen should help answer:

- What is this?
- Why does it matter?
- What is connected to it?
- What should I do next?

Preferred visualizations:

- Architecture layer diagrams.
- Dependency trees.
- Coverage heatmaps.
- Relationship tables.
- Domain maps.

Avoid:

- Force-directed graph visualizations.
- 3D graph clouds.
- Node-link hairball diagrams.
