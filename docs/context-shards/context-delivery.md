# Context Delivery Shard

This is a workflow and positioning evolution, not a product pivot.

Knowledge OS remains responsible for:

- Discovery.
- Extraction.
- Ontology.
- Graph.
- Reasoning.
- Context generation.
- Project generation.
- Assistant generation.
- Context delivery packaging.

Execution runtimes are responsible for:

- Collaboration.
- Editing.
- Execution.
- Iteration.
- Deliverables.

Target runtimes:

- ChatGPT Projects.
- Custom GPTs.
- SharePoint workspaces.
- Copilot agents.
- Markdown repositories.

Compiler model:

- Knowledge OS compiles context, ontology, dependencies, and knowledge assets.
- Runtimes consume those assets for reasoning, collaboration, and delivery.

Delivery outputs:

- Project space package.
- Context pack.
- Assistant instructions.
- Knowledge files.
- Architecture pack.
- Persona pack.

Acceptance rule:

Context delivery must preserve evidence, permissions, provenance, and runtime-specific constraints.
