# Wave Gates

Each wave must pass these gates before implementation can proceed or merge.

## Universal Gates

- Business goal trace exists.
- Relevant specs are approved.
- Security and performance review is complete.
- Public contracts have examples.
- Test and eval plan exists.
- Documentation shard set is defined.
- Independent business audit has no blocking findings.

## Wave 0 Gate

- Standards and sharding strategy exist.
- Audit rubric exists.
- Tooling baseline is verified.

## Wave 1 Gate

- Source acquisition and canonical document contracts are stable.
- Crawl security policy is defined.
- URL discovery and provenance tests exist.

## Wave 2 Gate

- Base ontology and Foundry domain pack are versioned.
- Extraction eval fixtures exist.
- Every candidate output requires evidence.

## Wave 3 Gate

- Canonical ID semantics are defined.
- Merge decisions are auditable.
- Graph writes are idempotent.

## Wave 4 Gate

- Retrieval authorization model is defined.
- Context assembly is bounded.
- Groundedness eval exists.

## Wave 5 Gate

- Context-pack schema is approved.
- Generated maps include evidence and confidence.
- Gap reports have acceptance criteria.

## Wave 6 Gate

- Project-space schema is approved.
- Assistant config deployment is separated from generation.
- Under-60-second project-space target has a measurement plan.

## Wave 7 Gate

- Context delivery contracts are approved.
- Export packaging preserves evidence, permissions, provenance, and runtime target metadata.
- Runtime-specific exporters do not fork the project-space or assistant source models.
- Export preview and receipt behavior is defined.

## Wave 8 Gate

- Top-level navigation remains limited to six sections.
- UX hides graph internals by default.
- UX applies the dark-first brand system.
- UX makes runtime export paths understandable where enabled.
- Primary workflow timing targets are measured.

## Wave 9 Gate

- Incremental update semantics are approved.
- Tenant and corpus permission boundaries are reviewed.
- Drift and change reports are auditable.
